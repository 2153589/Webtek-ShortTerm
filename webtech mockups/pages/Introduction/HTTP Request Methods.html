    HTTP Standard Request Methods
  • HTTP defines 8 standard methods, namely, GET, HEAD, POST, PUT, DELETE, OPTIONS, TRACE, and CONNECT, with GET and HEAD being required to be supported by HTTP-compliant servers

  • the behavior of the HTTP standard request methods can be characterized as follows:

  • safe: client does not request, and does not expect, any state change on the origin server as a result of applying the method to a target resource; i.e., the methodâ€™s effect is essentially read-only
  • GET, HEAD, OPTIONS, and TRACE are safe methods

  • idempotent: intended effect on the server of multiple identical requests with the method is the same as the effect of a single such request
  • GET, HEAD, OPTIONS, TRACE, PUT, and DELETE are idempotent methods

  • cacheable: indicates that the response to a method is allowed to be stored for future reuse
  • GET, HEAD, and POST are cacheable methods
